# 参考文章吸收说明

## 1. 来源

- [Agent 质量保障体系文章](https://mp.weixin.qq.com/s/GdTozBOtC-Add__fcGefXA?scene=1&click_id=7)
- [Eval 数据集建设文章](https://mp.weixin.qq.com/s/Hx4O9PthbLcVKvqLB7eyew?click_id=3)

## 2. 吸收原则

两篇文章不作为独立方法另起一套体系，而是补强现有 AI 评测 V1 的 5 个薄弱点：

- Eval 与传统测试用例的区别
- Eval 样本的真实来源与结构字段
- Agent 工具调用评估
- RAG 知识召回与问题诊断
- 发布前、发布时、发布后的质量保障分层

## 3. 已加入的内容

| 文章观点 | 加入位置 |
| --- | --- |
| Eval 不是传统测试用例升级版，应从真实问题和线上失败出发 | `framework/AI评测V1总框架.md` |
| 每条 Eval 至少包含输入/任务、预期行为、评估标准 | `framework/AI评测V1总框架.md`、`templates/黄金样本模板.md` |
| 样本来源要区分设计构造、线上翻车、线上发现、故障注入 | `templates/黄金样本模板.md`、`integration/AI评测接入清单.md` |
| Agent 需要检查工具名、参数、顺序、必要调用和多余调用 | `playbooks/Agent工作流操作手册.md` |
| RAG 要单独评估知识召回质量 | `playbooks/RAG问答操作手册.md` |
| 质量保障要分发布前、发布时、发布后和持续改进 | `framework/AI评测V1总框架.md`、`backlog/自动化与智能化待办清单.md` |

## 4. 对 V1 的影响

这些内容不会改变 V1 的定位：第一阶段仍然是“试点优化操作系统”，不直接做发布门禁。

它们带来的主要变化是：

- 黄金样本模板更适合后续结构化存储
- Agent 场景更强调工具调用和故障注入
- RAG 场景更容易区分检索问题与生成问题
- 自动化待办中新增线上问题入库、采样评估和 Guard 机制
