# AI评测接入清单

## 1. 用途

这份清单用于在新系统接入 AI 评测体系时快速确认：

- 现在缺什么
- 谁来补
- 是否已经达到可运行状态

## 2. 通用接入清单

### 2.1 基础识别

- [ ] 已识别能力类型：`RAG/问答` / `Agent 工作流` / `内容生成`
- [ ] 已指定主操作手册
- [ ] 已明确试点边界
- [ ] 已指定测试/质量负责人
- [ ] 已指定产品负责人
- [ ] 已指定工程/算法对接人

### 2.2 成功标准

- [ ] 已填写成功标准模板
- [ ] 已定义关键用户场景
- [ ] 已定义评估维度和权重
- [ ] 已定义及格线和优秀线
- [ ] 已明确本轮不覆盖范围

### 2.3 样本集

- [ ] 已收集首批真实问题/任务
- [ ] 已标注样本来源：设计构造 / 线上翻车 / 线上发现 / 故障注入 / 用户反馈
- [ ] 已标注案例类型：正常 / 边缘 / 混沌 / 记忆或上下文
- [ ] 已补齐证据来源或预期终态
- [ ] 已补齐可操作的评估标准
- [ ] 已标注难度标签
- [ ] 已标注 failure mode 标签
- [ ] 已筛出可进入正式评测的 approved 样本

### 2.4 运行入口

- [ ] 已有可执行入口
- [ ] 已确认输入输出格式
- [ ] 已确认结果能被保存或导出
- [ ] 已确认失败时可追溯

### 2.5 评分器

- [ ] 已定义基础规则检查
- [ ] 已定义 LLM judge rubric
- [ ] 如涉及工具调用，已定义工具名、参数、顺序、必要调用和多余调用检查
- [ ] 已定义人工抽检策略
- [ ] 已明确通过条件
- [ ] 已安排首轮校准

### 2.6 结果与复盘

- [ ] 已约好周节奏
- [ ] 已准备评测结果模板
- [ ] 已准备复盘模板
- [ ] 已确定复盘参会人
- [ ] 已定义行动项跟踪方式

## 3. 知识库接入专用清单

- [ ] 已确认知识域边界
- [ ] 已确认知识源可信且可追溯
- [ ] 已收集首批真实问题，建议 30 到 50 条
- [ ] 已确认每题对应的证据文档
- [ ] 已确认是否能拿到引用文档或检索结果
- [ ] 已确认是否能评估知识召回质量
- [ ] 已确认接入等级：L0 / L1 / L2
- [ ] 已定义首批评测维度：相关性、忠实度、完整性、清晰度、边界控制

## 4. Agent 接入专用清单

- [ ] 已确认任务集边界
- [ ] 已定义每类任务的成功终态
- [ ] 已确认可调用运行入口
- [ ] 已确认是否能拿到工具调用日志
- [ ] 已确认是否能拿到环境终态
- [ ] 已确认是否需要故障注入样本，例如工具超时、空数据、权限不足
- [ ] 已确认接入等级：L0 / L1 / L2
- [ ] 已定义首批评测维度：任务成功率、状态正确率、工具调用有效性、异常恢复能力

## 5. 接入等级判断

### L0：黑盒接入

- [ ] 只有输入和输出

### L1：灰盒接入

- [ ] 除输入输出外，还能提供至少一种中间信息
  - [ ] 检索结果
  - [ ] 引用文档
  - [ ] 工具调用日志
  - [ ] 中间状态

### L2：深度接入

- [ ] 有结构化轨迹
- [ ] 有环境终态
- [ ] 有批跑脚本
- [ ] 有结果沉淀方式

## 6. 接入完成判断

接入完成不要求先做平台化，至少需要满足：

- [ ] 已经选定主操作手册
- [ ] 已经完成成功标准
- [ ] 已经完成首批样本
- [ ] 已经定义评分器
- [ ] 已经跑出首轮评测结果
- [ ] 已经完成首轮复盘并形成行动项
