# AI Evals 文档导航

这份导航用于把 `AI Evals` 文档包收敛成一条容易进入的主线，避免第一次看的人不知道从哪篇开始。

## 1. 入门文件

第一次接触这套体系，建议先看下面两篇：

1. [AI评测V1总框架](./framework/AI评测V1总框架.md)
2. [AI评测体系建设说明（对外版）](./briefing/AI评测体系建设说明（对外版）.md)

对应作用如下：

- 这套体系的定位、边界、角色分工和运行闭环是什么
- 这项建设的背景、范围、建设内容和所需准备项

## 2. 整体结构地图

```text
AI Evals
├── framework      总框架：定义体系、术语、闭环、边界
├── integration    接入层：告诉已有系统怎么挂进来
├── playbooks      场景层：按能力类型说明“评什么、怎么评”
├── templates      执行层：统一结果、样本、评分器、复盘模板
├── pilot          试点层：首个试点的落地包
├── briefing       汇报层：对管理层和执行团队的同步材料
├── backlog        演进层：自动化和智能化待办
└── references     参考层：记录外部材料如何吸收到体系中
```

## 6. 结构说明

- `framework`：定义体系、术语、闭环和职责分工
- `integration`：说明已有系统如何接入评测
- `playbooks`：按能力类型说明评估重点和执行方式
- `templates`：统一样本、评分器、结果和复盘产物
- `pilot`：保留首个试点的完整落地材料
- `references`：记录参考资料如何进入当前标准，避免方法来源散落
