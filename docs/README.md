# AI 评测 V1 文档包

这套文档包用于帮助团队从 0 到 1 建立 AI 评测 V1 体系，并以 `内部知识问答` 作为首个试点场景跑通执行标准、运行节奏和复盘闭环。

- 接入入口：[AI评测接入指南](./integration/AI评测接入指南.md)
- Agent 测试入口：[Agent测试方法论Demo](./playbooks/Agent测试方法论Demo.md)

## 名词说明

- `操作手册`：原先常写作 `Playbook`，这里统一指“针对某一类 AI 能力的标准化执行手册”，说明它适用于什么场景、评什么、怎么接入、怎么运行。

## 推荐使用顺序

1. 先读 [文档导航](./文档导航.md)，确认当前要做的是体系理解、系统接入、试点启动，还是 Agent 测试。
2. 再阅读 [总框架](./framework/AI评测V1总框架.md)，对齐术语、边界和职责。
3. 如果是新系统要挂进来，继续看 [AI评测接入指南](./integration/AI评测接入指南.md) 和 [AI评测接入清单](./integration/AI评测接入清单.md)。
4. 根据业务类型选择对应操作手册。
   Agent 场景可同时参考 [Agent测试方法论Demo](./playbooks/Agent测试方法论Demo.md)。
5. 如果要启动第一批试点，直接使用 [内部知识问答试点包](./pilot/内部知识问答试点包.md)。
6. 在执行过程中复用 `templates/` 下模板，避免临时定义字段。
   需要沉淀到 Excel 看板或周报时，可直接复用 [模型评测结果模板（Excel版）](./templates/模型评测结果模板（Excel版）.md)。
7. 用 [汇报材料底稿](./briefing/AI评测V1汇报底稿.md) 同步管理层和执行团队。

## V1 边界

- V1 当前用于试点阶段的评估、复盘和优化协同，不承担上线门禁职责。
- 第一阶段以可复用、可执行、可复盘为主，后续再推进半自动化和智能化。
- V1 默认由 `测试/质量` 牵头，产品与工程共同参与。
