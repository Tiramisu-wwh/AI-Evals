# 内容生成操作手册

## 1. 适用场景

适用于以下类型的 AI 能力：

- 文案生成
- 摘要与改写
- 报告生成
- 风格化输出
- 品牌语气约束类生成

## 2. 评估对象

内容生成场景需要同时看任务目标和输出质量：

- 是否回答或满足目标任务
- 是否覆盖关键信息
- 是否表达自然、清晰、结构合理
- 是否符合品牌语气或风格要求
- 是否触发安全或合规问题

## 3. 核心指标

| 维度 | 说明 | 推荐 grader |
| --- | --- | --- |
| 帮助性 | 是否真正帮用户完成任务 | llm/human |
| 完整性 | 是否覆盖必须信息 | llm/human |
| 表达质量 | 是否清晰、流畅、结构合理 | llm |
| 风格一致性 | 是否符合品牌或目标风格 | llm/human |
| 安全合规 | 是否触发禁忌表达或风险 | code/human |

## 4. 推荐 grader 组合

- 规则检查：长度、禁词、格式、敏感内容
- LLM rubric：帮助性、完整性、表达质量、风格一致性
- 人工复核：用于高影响文案和 judge 校准

## 5. 试点启动步骤

1. 先缩小到一类典型任务，例如摘要或客服文案。
2. 用成功标准模板明确“好内容”的判断维度。
3. 选取代表样本并建立人工评分参考。
4. 编写分维度 rubric，避免单一总分掩盖问题。
5. 每轮复盘时重点看“哪类输出最不稳定”。

## 6. 复盘重点

- 是事实问题、风格问题还是结构问题
- judge 和人工是否对边界样本判断一致
- 模板、prompt 或 few-shot 是否足以解释波动
