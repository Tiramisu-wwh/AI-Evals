# 自动化与智能化待办清单

## 目标

本清单用于承接 AI 评测 V1 跑通后的后续建设，帮助团队从手工运营逐步过渡到半自动化和智能化。

## P0：先让试点可重复运行

| 项目 | 目标 | 输出 |
| --- | --- | --- |
| 样本结构化存储 | 把黄金样本从文档变成结构化数据 | 样本 schema、存储位置、字段规范 |
| grader 执行脚本 | 让 code grader 和 judge 可批量运行 | 批跑脚本、输入输出规范 |
| 结果汇总脚本 | 自动生成结果表和基础统计 | 分数汇总表、失败模式统计 |
| 周报生成 | 减少手工整理成本 | 标准周报产物 |
| 线上问题入库流程 | 把线上翻车、用户反馈和异常日志转成候选 Eval | 候选样本池、入库规则 |

## P1：提升质量团队运营效率

| 项目 | 目标 | 输出 |
| --- | --- | --- |
| judge 校准流程脚本化 | 固定人工校准方式 | 校准记录、差异报告 |
| failure mode 标签管理 | 统一标签变更和合并规则 | taxonomy 文档、变更机制 |
| 评测看板 | 可视化趋势变化 | 基础趋势看板 |
| 回归集沉淀 | 把稳定问题沉淀为回归基线 | 回归集清单 |
| 工具调用评估沉淀 | 将工具名、参数、顺序等检查项标准化 | tool_call_match 规则模板 |

## P2：智能化诊断

| 项目 | 目标 | 输出 |
| --- | --- | --- |
| 自动聚类 failure modes | 降低人工归类成本 | 聚类结果、建议标签 |
| 分数漂移告警 | 快速发现质量波动 | 告警规则、通知机制 |
| 根因推荐 | 基于样本和轨迹给出可能原因 | 根因建议卡片 |
| 回归集自动扩充建议 | 识别应加入回归集的新问题 | 候选回归样本 |
| 线上采样评估 | 从真实会话/任务中抽样并触发评估 | 采样规则、质量趋势 |

## P3：与发布和治理衔接

| 项目 | 目标 | 输出 |
| --- | --- | --- |
| 门禁候选规则筛选 | 识别哪些规则足够稳定可进入准入 | 候选规则列表 |
| 多团队复用机制 | 支持第二个、第三个团队接入 | onboarding 指南 |
| 平台化可行性评估 | 决定是否需要统一平台 | 平台方案草案 |
| 发布时 Guard 机制 | 对明显风险输入/输出做实时拦截或兜底 | 输入 Guard、输出 Guard、兜底策略 |

## 优先级原则

- 优先做能降低试点运营成本的自动化
- V1 阶段不把不稳定 judge 直接接到门禁
- 所有自动化都先继承文档中已冻结的字段和流程
