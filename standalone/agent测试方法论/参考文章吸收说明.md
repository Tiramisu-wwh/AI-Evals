## 2. 吸收原则

这 5 篇文章不单独形成另一套体系，只抽取其中可落地的测试做法，整理成单独的 Agent 测试专题材料。当前保留 9 类内容：

- 从需求评审阶段识别 Agent 任务边界和 ROI
- 从“测结果”扩展到“测约束、测状态、测过程”
- 用四层故障定位框架归因 badcase
- Eval 与传统测试用例的区别
- Eval 样本的真实来源与结构字段
- Agent 工具调用评估与状态快照
- LLM-as-Judge 与代码评分器的分工
- Multi-Agent 的节点、交接和调度分层
- 发布前、发布时、发布后的质量保障分层
