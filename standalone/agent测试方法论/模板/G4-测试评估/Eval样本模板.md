# Eval样本模板

## 1. 样本字段

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| case_id | 是 | 样本唯一编号 |
| 场景名称 | 是 | 样本标题 |
| 风险等级 | 是 | P0/P1/P2 |
| 来源 | 是 | 设计构造/线上翻车/线上发现/故障注入 |
| 类型 | 是 | 正常/边缘/混沌/记忆 |
| 用户输入 | 是 | 原始请求 |
| 前置状态 | 否 | 环境状态、上下文、附件 |
| 预期行为 | 是 | Agent 应做什么 |
| 禁止行为 | 是 | Agent 不得做什么 |
| 评分器 | 是 | code/state/tool/llm/human |
| 证据 | 否 | 日志、截图、state 快照 |
| 备注 | 否 | 说明 |

## 3. 样本池分层

### 3.1 smoke

- 5 到 10 条
- 覆盖主流程和 P0 风险

### 3.2 standard

- 20 到 50 条
- 覆盖主路径、边缘和已知失败模式

### 3.3 full

- 50 条以上
- 用于回归、灰度和专项评测

## 4. 维护规则

- 线上翻车样本修复后必须入库
- 每条样本至少对应一个明确评分器
- P0 样本不能只靠人工口头判断
- 同类问题连续出现两次，需要补同类型故障注入样本
